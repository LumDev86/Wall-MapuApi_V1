# Variables de Entorno

Este proyecto utiliza variables de entorno para configurar diferentes aspectos de la aplicación.

## Configuración

1. Copia el archivo `.env.example` y renómbralo a `.env`:
   ```bash
   cp .env.example .env
   ```

2. Actualiza las variables en el archivo `.env` con tus valores:

### Variables Disponibles

#### API Configuration
- `EXPO_PUBLIC_API_URL`: URL base de la API del backend
  - Desarrollo: `http://localhost:3000/api`
  - Producción: `https://wall-mapuapi-production.up.railway.app/api`

#### App Configuration
- `EXPO_PUBLIC_APP_NAME`: Nombre de la aplicación
- `EXPO_PUBLIC_APP_VERSION`: Versión de la aplicación

#### Environment
- `EXPO_PUBLIC_ENV`: Entorno de ejecución (`development`, `staging`, `production`)

## Notas Importantes

- **No subas el archivo `.env` a Git**: Este archivo contiene información sensible y ya está incluido en `.gitignore`
- **Usa el prefijo `EXPO_PUBLIC_`**: En Expo, solo las variables que comienzan con `EXPO_PUBLIC_` están disponibles en el cliente
- **Reinicia el servidor después de cambios**: Si modificas el archivo `.env`, necesitas reiniciar el servidor de Expo:
  ```bash
  npx expo start --clear
  ```

## Acceso en el Código

Las variables de entorno se acceden usando `process.env`:

```typescript
const apiUrl = process.env.EXPO_PUBLIC_API_URL;
```

## Ejemplo de Uso

```typescript
// src/services/api.ts
const API_URL = process.env.EXPO_PUBLIC_API_URL || 'https://default-api-url.com/api';
```
